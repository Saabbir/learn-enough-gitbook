## üß™ **What Is a Shopify Development Store?**

## üß∞ **Two Types of Development Stores**

Shopify offers **two distinct types of dev stores**, each designed for different use cases.

Choosing the correct type is important ‚Äî it prevents limitations later in your workflow.



### **1. Client Transfer Store**

Use this when you are building a real store for a client.

#### **Purpose**

* Set up and build fully functional client stores
* Transfer ownership when development is complete

#### **Key Features**

* Based on the latest Shopify release
* Can be upgraded into a real, paid store
* Includes most features from **Advanced** or **Plus** plans (depending on what you choose)

#### **Limitations**

* Cannot install **custom apps** or **draft apps**\
  (Use test and build stores for app development)
* Password page cannot be removed until ownership is transferred

#### **Use this when:**

‚úî You are building a store for an actual business\
‚úî You need to hand off ownership at the end\
‚úî A merchant will be running the store long-term



### **2. Development Store**

Use this when you‚Äôre testing features or building apps/themes.

#### **Purpose**

* Experiment with Shopify functionality
* Test apps, draft apps, or private apps
* Use developer preview features
* Build themes or custom storefronts for internal work

#### **Key Features**

* Created from the **Developer Dashboard**
* Can include **pre-generated demo data**:\
  products, orders, customers, etc.
* Supports custom apps, draft apps, and private apps

#### **Limitations**

* **Cannot be transferred** to a client
* Some storefront features remain restricted
* Real transactions cannot be processed

#### **Use this when:**

‚úî Building or testing apps\
‚úî Testing custom themes\
‚úî Experimenting with API functionality\
‚úî Trying new Shopify features before public release



## üîç **How to Choose the Right Dev Store Type**

Before creating a store, ask yourself one question:

> **Am I building this store for a client or for myself?**

#### If the store is for a client ‚Üí

**Use a Client Transfer Store**

#### If the store is for testing, experimenting, development, or learning ‚Üí

**Use a Development Store**

Choosing the correct type early saves time and prevents migration issues.



## ‚ö†Ô∏è **Development Store Limitations**

Dev stores are powerful but come with certain restrictions:

#### **General limitations**

* Only free or partner-friendly apps can be installed
* Only test orders can be created (using Bogus Test or Payment Test Mode)
* Real payment gateways cannot be used
* Password page cannot be removed
* Many Shopify Plus features may be restricted (unless using Plus dev store)

#### **Client transfer store limitations**

* Cannot install custom/draft apps
* Cannot use pre-generated data

#### **Test and build store limitations**

* Store cannot be transferred
* Pre-generated data makes the store permanently **non-transferable**
